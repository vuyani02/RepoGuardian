## Project Overview

This is an **ASP.NET Boilerplate (ABP) v9.4.1** full-stack application combining:
- **Backend**: ASP.NET Core 8 (`9.4.2/aspnet-core/`)
- **Frontend**: Angular 18 SPA (`9.4.2/angular/`)

Multi-tenancy, role-based authorization, JWT auth, and SignalR are all provided by the ABP framework.

## Commands

### Angular Frontend (`9.4.2/angular/`)

```bash
# Install dependencies
npm install

# Start development server (http://localhost:4200)
npm start
# or with Hot Module Replacement
npm run hmr

# Run tests
npm test

# Lint
npm run lint

# Production build
ng build --configuration production

# Regenerate API service proxies from backend Swagger
npx nswag run nswag.json
```

### ASP.NET Core Backend (`9.4.2/aspnet-core/`)

```bash
# Run the host
dotnet run --project src/FullStackProject.Web.Host

# Apply database migrations (run from solution root or EF project)
dotnet ef database update --project src/FullStackProject.EntityFrameworkCore --startup-project src/FullStackProject.Web.Host

# Run the migrator utility directly
dotnet run --project src/FullStackProject.Migrator

# Build solution
dotnet build

# Run tests
dotnet test
```

### Docker

```bash
# Run full stack (from 9.4.2/aspnet-core/docker/ng/)
docker-compose up
```

## Architecture

### Backend Layer Structure

```
FullStackProject.Core            → Domain entities, domain logic, constants
FullStackProject.Application     → Application services, DTOs, IAppService interfaces
FullStackProject.EntityFrameworkCore → EF Core DbContext, repositories, migrations
FullStackProject.Web.Core        → JWT config, base controllers, Swagger setup
FullStackProject.Web.Host        → Startup, Program.cs, appsettings.json, controllers
FullStackProject.Migrator        → Standalone DB migration runner
```

The backend uses **Castle Windsor** (not the default .NET DI container) registered via `UseCastleWindsor()`. ABP auto-registers services by convention — classes implementing `ITransientDependency`, `ISingletonDependency`, or `IApplicationService` are automatically registered.

**Adding a new feature**: create an entity in `Core`, an `IXxxAppService` + `XxxAppService` in `Application`, add DbSet to `EntityFrameworkCore`, create and apply a migration.

### Frontend Structure (`9.4.2/angular/src/`)

```
app/                 → Feature modules (lazy-loaded: home, about, users, roles, tenants)
shared/
  auth/              → AuthGuard, token service, credential storage
  service-proxies/   → Auto-generated API clients (do not edit manually)
  session/           → AppSessionService (current user/tenant info)
  layout/            → Layout store, sidebar/header components
  multi-tenancy/     → Tenant resolver (subdomain or query string)
  app-component-base.ts        → Base class for all components
  paged-listing-component-base.ts → Base class for paginated list pages
  AppConsts.ts       → Runtime constants (API URL, localization source name)
```

**App initialization flow**: `AppInitializerService` (in `app-initializer.ts`) runs before bootstrap — it loads `assets/appconfig.json`, calls `/AbpUserConfiguration/GetAll` to get permissions/localization/session, and resolves the current tenant.

**Routing**: All routes are guarded by `AppRouteGuard`. Permission-gated routes declare required permission in the route `data` (e.g., `data: { permission: 'Pages.Users' }`).

**API clients**: Generated by nswag from the backend Swagger spec into `shared/service-proxies/`. Re-run `npx nswag run nswag.json` after backend API changes — never edit these files manually.

**Base classes**: Extend `AppComponentBase` for all components (provides localization, permission checking, notify helpers). Extend `PagedListingComponentBase<T>` for paginated list views.

### Key Runtime Configuration

- Backend runs at `https://localhost:44311` (dev)
- Frontend runs at `http://localhost:4200` (dev)
- `assets/appconfig.json` sets `remoteServiceBaseUrl` and `appBaseUrl` at runtime (not compile time)
- Database: SQL Server `FullStackProjectDb` on localhost with Windows authentication (Trusted Connection)
- JWT secret is in `appsettings.json` under `Authentication:JwtBearer:SecurityKey`

### Real-Time (SignalR)

`AppComponent` initializes SignalR via `SignalRAspNetCoreHelper`. The ABP framework provides `abp.signalr.js` and `abp.signalr-client.js` (included as global scripts in `angular.json`).
